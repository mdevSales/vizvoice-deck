Why nothing has fixed this

Every "accessibility fix" assumes the goal is reading the chart.

Alt-text bandaids

Static one-line descriptions, written once by a sighted analyst. The data changes. The alt text doesn't.

Screen reader hell

Tableau renders as unlabeled SVG. VoiceOver announces "graphic, graphic, graphic" and gives up.

Tabular fallbacks

Data tables buried three clicks deep. Better than nothing. Worse than useful.

The reframe

What if you didn't
need to see the chart
at all?

The insight was never in the pixels. It was in the answer to the question you'd ask about them.

How it works

Ask the dashboard out loud.
Hear the answer back.

Grounded, not guessed

The agent reasons over the same semantic model as every chart in the org.

No screenshot parsing. No vision model. The agent queries the underlying Tableau semantic model directly via the Tableau REST API — so the data a blind user hears is the same authoritative source a sighted user sees.

Under the hood

Three deliberate layers
of design.

Layer 01 · The Agent
Speaks like a briefing, not a chart.
System prompt trained for accessibility first — numbers before adjectives, no color or shape references, every answer ends with a follow-up.
Layer 02 · The Interface
Keyboard-first, screen-reader complete.
WCAG 2.1 AA on every React component. No mouse required. Tested end-to-end with the same screen readers our users rely on.
Layer 03 · The Voice
Browser-native, zero cloud spend.
Web Speech for STT and TTS. Roughly half the compute per turn versus hosted speech pipelines — accessibility that is also sustainable.
